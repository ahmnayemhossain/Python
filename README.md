# 100 Days of Python

**For:** Serious Learner  
**Author:** A. H. M Nayem Hossain  
**Email:** nayem@softnan.com

---

## Learning Rules

- Complete **5 practice days** every week.
- Complete **1 weekly exam** every week.
- Take **1 full day off** every week.
- Build a small project every practice day, even if it is simple.
- Push every project to GitHub with a clean README file.
- Do not copy code blindly. Understand, write, debug, then improve.

---

## Daily Practice Format

```text
TODO: Review the previous topic for 15 minutes
TODO: Learn the new topic for 45 to 60 minutes
TODO: Solve 5 to 10 small exercises
TODO: Build the daily mini project
TODO: Write or update README.md
TODO: Commit and push to GitHub
```

---

## Weekly Pattern

| Day Type | Work | Time |
|---|---|---:|
| **Practice Day** | Concept, exercises, mini project, README, GitHub commit | 1.5 - 2.5 hours |
| **Weekly Exam** | Build without tutorial help and review weak areas | 2 - 3 hours |
| **Day Off** | Rest fully or do a light 20-minute review | Flexible |
